# CHANGELOG 0.4.5 — Parakeet TAGARELA + Mem0 + Host Mode simples

Base usada: ProjetoDiana 0.4.4 Query Gate / Operações Estruturais.
Referência usada: ProjetoDiana Mem0 Version, apenas para adaptar a arquitetura de Mem0 sem trazer a complexidade antiga.

## O que mudou

### 1. STT novo: Parakeet TAGARELA ONNX

Novo arquivo:
- stt/parakeet_stt.py

Configurações novas em config.py:
- STT_ENGINE = "parakeet"
- PARAKEET_MODEL_REPO = "alefiury/parakeet-tdt-0.6b-v3-ptBR-TAGARELA-onnx"
- PARAKEET_MODEL_PATH = "models/parakeet-tdt-0.6b-v3-ptBR-TAGARELA-onnx"
- PARAKEET_AUTO_DOWNLOAD = False por padrão
- PARAKEET_LANGUAGE = "pt"
- PARAKEET_ONNX_PROVIDER = "auto"

Whisper continua disponível como fallback:
- STT_ENGINE = "whisper"

O Parakeet trabalha com WAV temporário em 16 kHz mono e usa onnx-asr.

## Instalação sugerida do STT

pip install "onnx-asr[cpu,hub]" huggingface_hub

Baixar o modelo manualmente:

python -c "from huggingface_hub import snapshot_download; snapshot_download(repo_id='alefiury/parakeet-tdt-0.6b-v3-ptBR-TAGARELA-onnx', local_dir='models/parakeet-tdt-0.6b-v3-ptBR-TAGARELA-onnx')"

Ou ativar download automático:

set PARAKEET_AUTO_DOWNLOAD=true

## 2. Memória longa baseada em Mem0

Novo arquivo:
- brain/memory_mem0.py

Configurações novas em config.py:
- MEM0_ENABLED = True
- MEM0_USER_ID = "natan"
- MEM0_COLLECTION_NAME = "diana_memories"
- MEM0_QDRANT_PATH = "mem0_qdrant"
- MEM0_LLM_MODEL = LLM_MODEL
- MEM0_EMBEDDER_MODEL = "nomic-embed-text"
- MEM0_AUTO_SAVE_INTERACTIONS = False por padrão

Por padrão, a Diana busca Mem0 quando a pergunta é pessoal, de continuidade ou de lembrança.
Ela NÃO salva toda conversa automaticamente, para evitar transformar erro de STT ou zoeira de chat em memória.

Comando manual novo:
- /mem0 lembrar texto da memória
- /mem0 remember texto da memória

Exemplo:
/mem0 lembrar Natan prefere respostas curtas quando está testando código ao vivo.

## 3. PromptBuilder com Mem0 ancorado

Arquivo alterado:
- brain/prompt_builder.py

O prompt agora pode receber:
- FATOS PESSOAIS RECUPERADOS
- MEMÓRIAS MEM0 RECUPERADAS
- CONHECIMENTO RECUPERADO
- HISTÓRICO REAL DA SESSÃO

Regra importante:
- Mem0 entra como memória longa recuperada, não como pauta obrigatória.
- Base local continua tendo prioridade em perguntas técnicas já resolvidas.
- Se Mem0 não responder à pergunta pessoal, a Diana deve admitir que não sabe.

## 4. ContextRetriever com Mem0 leve

Arquivo alterado:
- brain/context_retriever.py

O Mem0 só é consultado em casos com sinais de memória pessoal/continuidade.
Isso evita que a memória longa contamine qualquer resposta casual.

## 5. Query Gate menos burro para humor e continuidade

Arquivo alterado:
- brain/query_gate.py

Correções:
- "conta outra piada" não vira consulta na base factual.
- "qual?", "como assim?", "por quê?" são tratados como continuidade curta quando adequado.

Objetivo: evitar o bug clássico:
Diana: Qual é a maior distância do mundo?
Natan: qual?
Diana: Qual pra quê?

Agora o prompt marca a tarefa como continuação direta do histórico real.

## 6. Host Mode com dois modos

Arquivo alterado:
- integrations/chat_host_mode.py

Config nova:
- HOST_MODE_KIND = "autonomous" ou "read_response"

Comandos novos:
- /host mode autonomous
- /host mode read
- /host read

Modo AUTÔNOMO:
- funciona como anfitriã, lendo novas mensagens por tick/cooldown.

Modo LEITURA E RESPOSTA:
- não roda sozinho no tick.
- responde apenas quando você pede /host read ou fala algo como "lê o chat e responde".

Envio ao chat continua separado:
- /host send on
- /host send off

## 7. O que NÃO foi trazido da 0.8/0.9

Não foi trazido:
- árvore grande de arquivos
- múltiplos sistemas competindo pelo mesmo contexto
- memória automática agressiva
- remendos específicos por palavra
- arquitetura que tenta resolver tudo ao mesmo tempo

A 0.4.5 mantém a linha da 0.4.4:
- fluxo principal simples
- Query Gate como filtro
- base local como fonte factual
- histórico real para continuidade
- Mem0 como camada de memória longa, sem atropelar o resto

## Teste rápido

Rodar:

python test_retrieval_0_4_5.py

O teste valida:
- piada não ativa consulta factual
- follow-up curto sem entrada ativa não consulta base
- follow-up técnico com entrada ativa consulta base
- Mem0 entra no prompt como contexto ancorado
- "qual?" vira tarefa de continuidade curta
