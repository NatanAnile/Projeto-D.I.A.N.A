# DIANA 0.4 — MODELO AUXILIAR

A versão 0.4 já possui a estrutura de dois modelos auxiliares:

1. Query Planner
   - Interpreta somente a consulta.
   - Não responde como Diana.
   - Produz um plano JSON para o KnowledgeRetriever.

2. Session Summarizer
   - Resume lotes de turnos antigos.
   - Mantém as últimas interações completas.
   - Salva o resumo em:
     data/context/episodic_summary.json
     data/context/episodic_summary.txt

POR PADRÃO, OS DOIS ESTÃO DESATIVADOS.
Nenhum segundo modelo é carregado e nenhuma chamada extra é feita.

## Quando for ativar

No config.py:

AUXILIARY_LLM_URL = "http://localhost:11435/api/generate"
AUXILIARY_LLM_MODEL = "qwen2.5:1.5b-instruct"
QUERY_PLANNER_ENABLED = True
SESSION_SUMMARIZER_ENABLED = True

Você pode ativar somente um deles.

## Segunda instância do Ollama

Execute:

tools/start_aux_ollama_cpu.bat

A instância auxiliar usa a porta 11435. O script tenta ocultar a GPU com
CUDA_VISIBLE_DEVICES=-1 para priorizar CPU. Confirme com:

ollama ps

O campo PROCESSOR deve indicar 100% CPU para o modelo auxiliar.

## Carregamento sob demanda

O cliente auxiliar envia keep_alive=0. Assim, após cada chamada, o Ollama
recebe a instrução para descarregar o modelo auxiliar imediatamente.

## Resumidor

O resumidor só é chamado quando:

SESSION_SUMMARIZER_ENABLED = True

e o número de turnos pendentes alcança SESSION_SUMMARIZER_TRIGGER_TURNS.

A chamada acontece em thread separada. A conversa principal continua usando
o histórico recente enquanto o resumo é gerado.
