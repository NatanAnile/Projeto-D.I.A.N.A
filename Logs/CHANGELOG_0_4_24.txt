# Diana 0.4.24 — FEEDBACK_BOUNDARY_REPO_CI

## Objetivo

Fechar a fronteira entre crítica negativa à última fala da Diana e correção factual do Neitan, além de começar a transformar o repositório em uma base mais limpa para publicação nova.

## Mudanças principais

- `feedback_negative` foi conceitualmente expandido para `feedback_negative_previous_response`.
- Novo ato `factual_correction` com prioridade acima de feedback negativo.
- Correções factuais como "você errou sobre Draygon" não caem mais em deboche de piada ruim.
- Críticas de qualidade como "sua resposta ficou ruim" continuam sendo tratadas como crítica à última fala da Diana.
- Respostas diretas foram movidas para `personality/dialogue_response_bank.py`.
- Banco de respostas ampliado para reduzir repetição em live.
- Criado workflow `.github/workflows/tests.yml` para GitHub Actions.
- README atualizado para 0.4.24 e teste atual.
